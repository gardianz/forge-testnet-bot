# Forge Testnet Bot Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a multi-account, idempotent bot that runs the Forge Lending testnet checklist (substrate faucet → bridge → EVM faucet → warp → supply → borrow → repay → redeem) on Bittensor EVM chain 945.

**Architecture:** Hybrid. Playwright drives only the captcha-gated substrate faucet; everything else is on-chain (viem for EVM, polkadot.js for substrate) or a direct HTTP call. Each step self-checks on-chain state and skips when its goal is already met, so runs are resumable and safe to repeat. A `runner` processes accounts concurrently behind per-account proxies.

**Tech Stack:** TypeScript (ESM), `tsx`, `vitest`, `viem`, `@polkadot/api` + `@polkadot/keyring` + `@polkadot/util-crypto`, `playwright`, `zod`, `pino`, `node-cron`, `undici`, `socks-proxy-agent`, `crypto-js`.

## Global Constraints

- **Network:** Bittensor EVM testnet only. EVM RPC `https://test.chain.opentensor.ai`, chainId `945`. Substrate RPC `wss://test.finney.opentensor.ai:443`, ss58Format `42`. Any write path MUST assert `chainId === 945` and refuse otherwise.
- **Module system:** ESM (`"type": "module"`), TypeScript strict, `import`/`export` only.
- **Secrets:** never logged. `accounts.json`, `config.yaml`, `.env`, `state/`, `logs/` are gitignored.
- **Derivation:** EVM = secp256k1 path `m/44'/60'/0'/0/0`. Substrate = sr25519, ss58Format 42.
- **Defaults to safe:** `dryRun` defaults true; broadcasting requires explicit `dryRun: false` or `--no-dry-run`.
- **TDD:** write the failing test first, watch it fail, implement minimal, watch it pass, commit. Pure logic (derivation, skip-checks, encoding, schemas) is unit-tested with mocks; live chain calls are integration-tested behind an env gate (`FORGE_LIVE=1`).
- Address constants are **provisional** until Task 6 verifies them against the live chain. Tasks 9–16 MUST read the verified address map from `src/contracts.ts`, never hardcode.

import CryptoJS from 'crypto-js';

export function encryptJson(obj: unknown, key: string): string {
  return CryptoJS.AES.encrypt(JSON.stringify(obj), key).toString();
}
export function decryptJson(cipher: string, key: string): unknown {
  const bytes = CryptoJS.AES.decrypt(cipher, key);
  return JSON.parse(bytes.toString(CryptoJS.enc.Utf8));
}

import { readFileSync } from 'node:fs';
import { decryptJson } from './crypto.ts';

export type Account = {
  id: string; h160: `0x${string}`; ss58: string;
  mirrorSs58: string; mnemonic: string; proxy?: string;
};

export async function loadAccounts(path: string, key: string, ss58Format: number): Promise<Account[]> {
  const text = readFileSync(path, 'utf8');
  // Support either an encrypted blob (string) or plain JSON array (dev).
  const raw = text.trim().startsWith('[') ? JSON.parse(text) : decryptJson(text, key);
  const list = raw as Array<{ id: string; mnemonic: string; proxy?: string }>;
  return list.map((a) => ({
    id: a.id, mnemonic: a.mnemonic, proxy: a.proxy || undefined,
    h160: deriveH160(a.mnemonic), ss58: deriveSs58(a.mnemonic, ss58Format),
    mirrorSs58: '',
  }));
}

```typescript
import { blake2AsU8a, encodeAddress } from '@polkadot/util-crypto';
import { hexToU8a, stringToU8a, u8aConcat } from '@polkadot/util';

export function h160ToMirrorSs58(h160: string, ss58Format: number): string {
  const addr = hexToU8a(h160.startsWith('0x') ? h160 : '0x' + h160);
  const data = u8aConcat(stringToU8a('evm:'), addr);
  const hashed = blake2AsU8a(data, 256);
  return encodeAddress(hashed, ss58Format);
}
```

import { readFileSync, writeFileSync, mkdirSync, existsSync } from 'node:fs';

export type StepStatus = 'pending' | 'done' | 'skipped' | 'failed';
export type AccountState = { id: string; steps: Record<string, { status: StepStatus; tx?: string; at?: string; error?: string }> };

const dir = 'state';
const file = (id: string) => `${dir}/${id}.json`;

export function loadState(id: string): AccountState {
  if (!existsSync(file(id))) return { id, steps: {} };
  return JSON.parse(readFileSync(file(id), 'utf8'));
}
export function saveStep(id: string, step: string, patch: Partial<AccountState['steps'][string]>): void {
  mkdirSync(dir, { recursive: true });
  const s = loadState(id);
  s.steps[step] = { status: 'pending', ...s.steps[step], ...patch, at: new Date().toISOString() };
  writeFileSync(file(id), JSON.stringify(s, null, 2));
}
